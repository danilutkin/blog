Understanding iOS Deep Links for VLESS Imports

Deep links let iOS hand off URLs directly to apps that have registered specific schemes. When the user taps a link with the streisand:// or v2box:// scheme, the operating system looks for a matching app identifier and, if present, launches the app while passing everything after the scheme as parameters.

Both Streisand and V2box register custom URL schemes to streamline importing proxy configurations. The apps expect a JSON or base64 payload that matches the VLESS format, so it is common to wrap the configuration in a single URL-encoded parameter. Below are two example deep links that target each app and deliver the same VLESS configuration:

  1. Streisand deep link: streisand://import?config=%7B%22v%22%3A%222%22%2C%22ps%22%3A%22Example%20VLESS%22%2C%22add%22%3A%22example.com%22%2C%22port%22%3A443%2C%22id%22%3A%22d3d94468-2e02-4a02-b1f5-8da3eb3b0123%22%2C%22aid%22%3A0%2C%22scy%22%3A%22auto%22%2C%22net%22%3A%22ws%22%2C%22type%22%3A%22none%22%2C%22host%22%3A%22cdn.example.com%22%2C%22path%22%3A%22%2Fvless%22%2C%22tls%22%3A%22tls%22%7D
  2. V2box deep link: v2box://import?config=%7B%22v%22%3A%222%22%2C%22ps%22%3A%22Example%20VLESS%22%2C%22add%22%3A%22example.com%22%2C%22port%22%3A443%2C%22id%22%3A%22d3d94468-2e02-4a02-b1f5-8da3eb3b0123%22%2C%22aid%22%3A0%2C%22scy%22%3A%22auto%22%2C%22net%22%3A%22ws%22%2C%22type%22%3A%22none%22%2C%22host%22%3A%22cdn.example.com%22%2C%22path%22%3A%22%2Fvless%22%2C%22tls%22%3A%22tls%22%7D

In both cases the config parameter contains a URL-encoded JSON document. Decoded, it looks like this:

{ "v": "2", "ps": "Example VLESS", "add": "example.com", "port": 443, "id": "d3d94468-2e02-4a02-b1f5-8da3eb3b0123", "aid": 0, "scy": "auto", "net": "ws", "type": "none", "host": "cdn.example.com", "path": "/vless", "tls": "tls" }

Because iOS treats deep links as normal URLs, they can be shared via Messages, Notes, or QR codes. When the Streisand or V2box apps are installed, tapping these links immediately opens the app, parses the payload, and adds the VLESS server to the user’s list. If the app is not installed, iOS instead prompts the user to download it from the App Store.

For a smoother experience, ensure the deep link is percent-encoded, keep the JSON payload concise, and test the link on a device with the target app installed. That way users can reliably import the VLESS configuration with a single tap, regardless of which client they prefer.
